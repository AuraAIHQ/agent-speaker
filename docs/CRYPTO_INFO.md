# Nostr 加密算法说明

## 当前使用的算法

### 1. 身份密钥（无法更改 - 协议规定）
```
算法: secp256k1 (椭圆曲线)
私钥: 32 字节随机数
公钥: 33 字节压缩格式 / 65 字节未压缩
签名: Schnorr 签名 (BIP-340)
```

**这是 Nostr 协议的核心，所有客户端必须使用 secp256k1**
- 与比特币、以太坊使用相同曲线
- 无法改为 RSA、Ed25519 或其他曲线
- 如果更改，就不是 Nostr 协议了

### 2. 端到端加密（NIP-44）
```
密钥交换: ECDH (secp256k1)
对称加密: XChaCha20-Poly1305
```

## 用户提到的算法

### "RR one" - 可能指 RSA?
```
RSA: 2048/4096 位
特点: 基于大质数分解
与 Nostr: 不兼容
```

### "secp156" - 可能打错了?
可能的含义:
- **secp256k1**: 比特币/Nostr 正在使用的
- **secp256r1** (aka P-256): 美国政府标准
- **secp384r1**: 更强但较慢
- **Curve25519** (X25519): 现代、快速、安全

---

## 结论

**Nostr 协议强制使用 secp256k1，无法更改**

如果用户想要：
1. **更强的安全性**: 使用 NIP-44 E2E 加密（在应用层加密消息内容）
2. **不同的算法**: 需要放弃 Nostr，使用其他协议（如 Matrix/Signal）
3. **抗量子**: 目前 Nostr 没有抗量子方案
